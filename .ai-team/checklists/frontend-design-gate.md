# Frontend Design Gate

Use before backend API design or core implementation for user-facing products.

## Required Design

- [ ] Page map or screen map.
- [ ] Primary user journey.
- [ ] Core screens or wireframes.
- [ ] Component inventory: buttons, cards, forms, tables, dialogs, navigation.
- [ ] Loading, empty, error, and success states.
- [ ] Mobile/responsive behavior when relevant.
- [ ] Visual direction that matches the product audience and use case.

## Rules

- Do not design backend APIs until frontend interactions are known.
- Every backend endpoint should trace to a visible user action, scheduled job, integration, or admin operation.
- For new products, prefer a usable first screen over a marketing landing page unless the product is explicitly a public website.
- If the product depends on visual appeal, get Human Lead approval on the frontend direction before core backend implementation.

## Output

- screen list
- key components
- interaction list
- state list
- frontend acceptance criteria
