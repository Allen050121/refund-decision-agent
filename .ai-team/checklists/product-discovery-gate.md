# Product Discovery Gate

Use before technical planning or coding for a new product, large feature, or unclear user request.

## Must Know

- [ ] Target users and buyer/user difference when relevant.
- [ ] Primary pain point and current workaround.
- [ ] Top 1 to 3 MVP use cases.
- [ ] Core retention feature: what makes users come back.
- [ ] Non-goals for the first version.
- [ ] Success criteria for the first usable release.

## Ask When Missing

- Ask 1 to 3 concise questions when audience, MVP boundary, or success criteria are unclear.
- Recommend a simple default when the user wants speed.
- Do not choose a stack, scaffold code, or design APIs before this gate has enough answers.

## Output

Record a compact product brief:

- audience
- problem
- MVP scope
- non-goals
- success criteria
- open decisions
