# Quality Gate Checklist

Use before marking an implementation task done.

- [ ] Change is inside task boundaries.
- [ ] Code follows existing project patterns.
- [ ] No unnecessary dependency, abstraction, or framework was added.
- [ ] User-facing paths include error, loading, and empty states when relevant.
- [ ] Names are clear and modules stay small.
- [ ] Security-sensitive data is not leaked or committed.
- [ ] Verification command ran, or waiver is documented.
- [ ] Handoff notes explain risks and follow-ups.
