# Plan Gate Checklist

Use before assigning tasks to Executors.

- [ ] Goal and success criteria are clear.
- [ ] Non-goals are stated for risky or broad work.
- [ ] Tasks are split by file boundary and dependency, not by job-title role.
- [ ] Parallel tasks do not modify overlapping files.
- [ ] Shared schemas, auth, payment, migrations, build config, and common APIs are serial unless explicitly justified.
- [ ] Each task has acceptance criteria.
- [ ] Each task has at least one verification command or explicit verification waiver.
- [ ] Each Executor can start from memory summary, task card, and relevant files only.
- [ ] Planned parallelism is 2 to 3 agents by default and no more than 4.
